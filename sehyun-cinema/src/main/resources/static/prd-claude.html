세현시네마 (SEHYUN CINEMA) — PRD 요약본

이 문서는 발표용 PPT 슬라이드를 다른 AI(Claude 등)에 넣을 때 복사·붙여넣기 하기 쉽게 만든 제품 요약서입니다. 실제 서비스 코드는 Spring Boot 프로젝트 sehyun-cinema에 있습니다.

접속 URL
로컬 서버 기동 후 브라우저: http://localhost:8080/ (예매 사이트)
이 PRD 페이지: http://localhost:8080/prd-claude.html

1. 제품 개요

  • 이름: 세현시네마 (SEHYUN CINEMA)
  • 유형: 온라인 영화관 예매·멤버십 웹 서비스 (롯데시네마·CGV 등 멀티플렉스 UX 참고)
  • 목표: 영화 선택 → 상영관·일시·좌석 → 결제까지 한 흐름, VIP 등급·쿠폰·가상카드 할인, TMDB 포스터로 박스오피스 영화 시각화
  • 기술: Java 21, Spring Boot 3.4.x, Thymeleaf, Spring Security, OAuth2(카카오), PostgreSQL(Supabase), Gradle

2. 핵심 기능

  • 메인: 박스오피스 TOP10, 현재상영·개봉예정·아르떼 탭, 영화 검색·더보기, TMDB 포스터
  • 영화 상세: 포스터, 설명, 평점·예매율, 찜(하트), 예매하기, 관람평(별점 1~5 + 텍스트)
  • 예매: 영화·상영관·일시·인원·좌석 → 결제(가상카드·간편결제 등), 예매 완료 화면
  • 회원: 일반/VIP/VVIP/LVIP 등급, 누적 결제 금액 기준, 마이페이지·예매내역·쿠폰
  • 기타: 공지·FAQ·1:1 문의, AI 상담톡(API), 스페셜관 안내(일반·IMAX·4DX·수퍼플렉스·리클라이너 등)

3. 포스터·데이터

  • 영화 포스터 URL은 TMDB 이미지(image.tmdb.org)를 DB에 저장해 사용
  • 기동 시 제목과 맵이 일치하면 포스터 URL 동기화, 빈·플레이스홀더 URL은 보정

4. 보안·계정

  • 폼 로그인 + 카카오 OAuth2
  • 예매·마이페이지 등은 로그인 필요
  • 데모 계정은 발표 시 안내용으로만 사용 (실서비스 비밀번호는 노출 금지)

5. PPT용 메시지 (슬라이드 후보)

  • 슬라이드 1: 제목 — 세현시네마 PRD / 서비스 기획 및 개발 발표
  • 슬라이드 2: 문제 정의 — 기존 예매 UX의 복잡성, 멤버십 혜택 가시화 필요
  • 슬라이드 3: 솔루션 — Spring Boot 기반 통합 예매 + 등급제 + TMDB 연동
  • 슬라이드 4: 아키텍처 — Web / Security / JPA / PostgreSQL
  • 슬라이드 5: 데모 시나리오 — 메인 → 영화 → 예매 → 결제
  • 슬라이드 6: 마무리 — Q&A, 연락처
